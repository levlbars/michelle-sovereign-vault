Architecture

Two-graph split, never rejoined at inference. Tower and LM communicate via projected embeddings only.

Language Model (W8A16 · Hexagon NPU)
  • 32 layers, hidden_size=2560, vocab=248,320
  • Split into 5 ONNX chunks → 5 HTP context binaries
  • INT8 weights, FP16 activations
  • W8A16 encoding patch covers all self-attention MatMul inputs (methodology gated)
  • Gate-passed FP32 bit-exact via constant-fold (onnxslim)
Vision Tower (FP16 · Adreno GPU)
  • FP16 — not quantized (intentional: 8-bit introduces noisy embeddings)
  • Frozen during SFT training
  • Communicates with LM via projected embeddings at <|vision_start|>..<|vision_end|> slots
  • Runs on Adreno GPU for continuous monitoring use cases
Design principle: The tower and LM are two graphs forever — they never merge at inference. Keeping the tower at FP16 preserves embedding quality that W8A16 quantization would degrade.

Intended Use & Limitations

On-device multimodal reasoning on Snapdragon 8 Gen 3 devices. Target: continuous visual monitoring, on-device reasoning without cloud dependency, privacy-sensitive multi-domain inference.

  • Optimized for SM8650 — HTP context binaries are device-specific to Snapdragon 8 Gen 3.
  • Prefill benchmark is measured; decode throughput pending decode-shaped [1,1,2560] graph compile.
  • Per-run context reload overhead present — persistent-context runner will improve end-to-end latency.
  • Vision tower fine-tuning not included in current SFT (text-only data, tower frozen).
